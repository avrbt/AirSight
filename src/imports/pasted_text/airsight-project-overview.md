1. Project Overview & ArchitectureAirSight is an AI-powered Air Quality Index (AQI) tracking, forecasting, and pollutant source attribution platform. It combines real-time GIS spatial maps, machine learning predictive modeling, and user alert systems into a unified dashboard.Core Tech Stack MatrixDesign/Frontend: React, TypeScript, Tailwind CSS, Vite, Zustand (State Management).Backend Application: Python, FastAPI, Alembic (Database Migrations), SQLAlchemy ORM.Database & GIS: PostgreSQL with PostGIS extensions (implied by spatial features and ward boundaries).AI/Machine Learning: XGBoost (tabular/spatial feature forecasting), LSTM (temporal/sequential forecasting), SHAP (Explainable AI for pollutant attribution).Infrastructure: Docker, Nginx, GitHub Actions (CI/CD).2. Stage A: Ingestion & Spatial ContextualizationIngestion: The backend background services (external.py) execute scheduled requests fetching telemetry from environmental sensor networks and external atmospheric APIs.Spatial Mapping: Incoming raw geo-coordinates pass through the GIS service layer (gis.py) which maps coordinates to defined geographic areas or city divisions (ward.py model).Stage B: Storage, State, & Data SchemaData Processing: The ingested data is processed via data models (aqi.py, user.py, alert.py).Persistence: Structured entries are written to the database through CRUD utilities (aqi.py, alerts.py), operating via session.py DB contexts.Stage C: Machine Learning Pipeline & Inference FlowFeature Extraction: Whenever a forecast or evaluation is queried, the pipeline passes the raw timeline through custom feature modules:spatial.py: Extracts regional features, proximity calculations, and node distributions.temporal.py: Processes time-of-day variables, seasonal impacts, and lagging values.Model Selection: The extracted features feed directly into the predictive engines:predict.py executes inferences using train_xgboost.py (for instant spatial matrix projections) or train_lstm.py (for deep chronological sequences).Explainability Analysis: To provide source attribution, the input values pass through explain.py, which generates feature impact scores via a SHAP framework.Stage D: API Delivery to Frontend RouterThe processed data is serialized using custom schemas (schemas/) and served instantly via core FastAPI endpoints:GET /api/v1/aqi/: Delivers current contextualized metrics to useFetchAQI.ts.GET /api/v1/forecast/: Streams time-series predictive paths directly into the interactive ForecastChart.tsx.GET /api/v1/attribution/: Provides mathematical weighting matrices to populate the AttributionDonut.tsx and SHAPForcePlot.tsx structures.POST /api/v1/alerts/: Saves alert configurations, checked against automated scripts running via services/alerts.py.4.
